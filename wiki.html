📌 О проекте

Sputnik Studio — веб-приложение для проектирования AV-систем. Позволяет инженерам и проектировщикам:

  • Создавать и управлять проектами (бюджет, статусы, дорожные карты).
  • Рассчитывать тракты передачи видео/аудио (задержки, битрейт, PoE).
  • Выполнять специализированные расчёты: LED-экраны, звуковое давление (SPL), ВКС, эргономика, питание.
  • Генерировать техническую документацию на основе шаблонов.

Версия v8 — полная перезапись на React с TypeScript, использующая современные подходы: компонентная архитектура, централизованное состояние (Redux Toolkit), роутинг (React Router) и хранилище IndexedDB для данных.

🏗 Архитектура

Проект построен по модульному принципу. Основные блоки:

src/
├── assets/            # статика (шрифты, иконки)
├── components/        # переиспользуемые UI-компоненты
│   ├── common/        # кнопки, модалки, поля ввода
│   ├── layout/        # Topbar, Sidebar
│   └── widgets/       # виджеты дашборда
├── features/          # фичи (калькуляторы, тракты, проекты)
│   ├── tracts/
│   ├── projects/
│   ├── video/
│   ├── network/
│   ├── led/
│   ├── sound/
│   ├── vc/
│   ├── ergo/
│   └── power/
├── hooks/             # кастомные хуки
├── pages/             # страницы (роутинг)
├── store/             # Redux store и слайсы
├── styles/            # глобальные стили и CSS-переменные
├── types/             # TypeScript типы
├── utils/             # утилиты, modelDB, расчёты
├── App.tsx
└── main.tsx

Роутинг — React Router v6. Четыре основных маршрута: /dashboard, /projects, /calculations, /templates. Каждый маршрут монтирует соответствующую страницу, полностью заменяя содержимое <main>.

Состояние — Redux Toolkit. Слайсы разделены по функциональности: theme, ui, video, network, tracts, projects (и далее для каждого калькулятора). Все действия и редьюсеры типизированы.

Хранилище данных — IndexedDB через библиотеку Dexie. Проекты и тракты сохраняются локально, поддерживается экспорт/импорт JSON.

🧩 Функциональные модули

📁 Проекты

  • Создание/редактирование/удаление проектов.
  • Поля: название, бюджет, инженер, руководитель проекта, дата начала, статус (пресейл → стадия П → стадия Р → монтаж → завершён).
  • Дорожная карта: автоматический прогресс в зависимости от статуса, возможность указать дату следующего этапа.
  • Встречи и закупки (динамические списки).
  • Приоритетные проекты (отмечаются флагом).

🔧 Тракты (AV‑тракты)

  • Создание нескольких независимых трактов (путей прохождения сигнала).
  • Добавление устройств из базы моделей (источники, передатчики, матрицы, приёмники, дисплеи, коммутаторы, LED‑процессоры).
  • Автоматический расчёт: суммарная задержка, общая мощность, битрейт, использование портов коммутаторов, PoE‑бюджет.
  • Отображение активного тракта или всех трактов в виде списка.

📺 Видео

  • Настройки: разрешение (1080p/4K/8K), субдискретизация (4:4:4/4:2:2/4:2:0), частота кадров, цветовое пространство, битность.
  • Расчёт битрейта на основе выбранных параметров.

🌐 Сеть

  • Выбор среды передачи (Cat5e–Cat8, оптика, беспровод).
  • Настройки Multicast, QoS, типа сети (управляемая/неуправляемая), протокола синхронизации (PTP/NTP), резервирования.
  • Статистика: суммарный битрейт трактов, загрузка канала, количество использованных портов, статус Multicast/QoS.

💡 LED‑калькуляторы

  • По кабинетам: размеры кабинета, шаг пикселя, количество кабинетов → расчёт разрешения, площади, мощности.
  • По разрешению: целевое разрешение экрана, шаг пикселя → расчёт размеров и площади.

🔊 Звуковые калькуляторы

  • SPL на расстоянии, падение SPL от расстояния, изменение SPL от мощности.
  • RT60 (реверберация) по объёму помещения и коэффициенту поглощения.
  • Подбор громкоговорителей (мощность, чувствительность, требуемый SPL).

📡 ВКС (видеоконференцсвязь)

  • Расчёт битрейта кодеков (TrueConf, WebRTC, H.264, H.265) в зависимости от разрешения, FPS и количества участников.
  • Многоточечный вызов: расчёт нагрузки на MCU.

🪑 Эргономика ЭКП

  • Калькулятор углов обзора, высоты установки, расстояния до экрана для электронных классных панелей.

⚡ Питание

  • Расчёт ИБП (автономность, мощность).
  • Расчёт кондиционирования (тепловыделение оборудования, требуемая мощность охлаждения).

📄 Шаблоны документов

  • Генерация пояснительной записки, спецификации оборудования, акта выполненных работ.
  • Выбор подсистем проекта для включения в документ.
  • Привязка к конкретному проекту (данные подставляются автоматически).

🎨 Дизайн и стили

Используется система CSS-переменных (кастомные свойства) для светлой и тёмной тем. Переменные определены в src/styles/variables.css.

Цветовая палитра (светлая тема)

  • --accent: #2563eb (основной акцентный синий)
  • --bg-page: linear-gradient(135deg, #f6f9fc 0%, #edf2f9 100%)
  • --bg-panel: rgba(255,255,255,0.85) (стеклянная панель)
  • --text-primary: #1a2c3e
  • --text-secondary: #4b6a8a
  • --danger: #dc2626

Тёмная тема переопределяет переменные через селектор body.dark.

Шрифты

  • Основной: Inter, запасные – -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui.
  • Моноширинный (для кода): 'Fira Code', monospace.

Компоненты

  • Карточки (.project-card, .stats-card) со стеклянным фоном, скруглёнными углами (32px) и тенью.
  • Кнопки: .btn-primary (градиентный синий), .btn-secondary (обводка), .btn-danger (красный).
  • Поля ввода, селекты – скруглённые, с эффектом фокуса.
  • Аккордеоны в сайдбаре – сворачиваются, сохраняют состояние в localStorage.

🚀 Инструкция по запуску

Требования: Node.js 18+ и npm (или yarn/pnpm).
  1. Клонируйте репозиторий (или получите файлы проекта):
    git clone https://github.com/rawcam/sputnik-studio-react.git
    Если репозиторий ещё не создан, создайте его и скопируйте в него все файлы, которые я предоставил.
  2. Установите зависимости в корневой папке:
    npm install
  3. Запустите сервер разработки:
    npm run dev
    Приложение откроется по адресу http://localhost:3000 (порт может быть другим, если 3000 занят).
  4. Сборка для продакшена:
    npm run build
    Готовая статика появится в папке dist/.

Настройка хранилища IndexedDB

При первом запуске в браузере будет создана база данных SputnikDB с таблицами projects и tracts. Данные сохраняются автоматически при изменениях.

Импорт данных из старой версии

В панели управления есть кнопки «Экспорт JSON» и «Импорт JSON». Можно перенести проекты и тракты из предыдущей версии (v7), экспортировав их в JSON и загрузив в v8.

Важно: Для корректной работы сайдбара (показывается только на странице расчётов) используется условие в Sidebar.tsx. Если вы хотите показывать сайдбар на всех страницах, измените логику в компоненте.
